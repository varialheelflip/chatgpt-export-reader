# ChatGPT Export Reader

一个用于本地阅读 ChatGPT 导出 JSON 聊天记录的轻量工具，支持会话列表浏览、Markdown 渲染、分支切换，以及导入 ZIP 和删除会话文件。

## 背景说明

这个项目用于阅读 Chrome 插件 [ChatGPT Degrade Checker - 降级检测器](https://chromewebstore.google.com/detail/chatgpt-degrade-checker-%E9%99%8D/inidgeckbobnafenlmlgfbeoijiamepm) 导出的 JSON 聊天记录。

导出的聊天记录包含树状分支结构，普通 JSON 查看方式不方便阅读，也不方便在不同分支之间切换。本项目的目标是把这些 JSON 记录整理成更接近日常聊天界面的阅读体验，并支持在分支节点之间切换查看。

## 功能特性

- 读取指定文件夹中的 JSON 聊天记录
- 支持导入 ZIP 文件，自动提取根目录下的 `.json` 文件到当前 JSON 文件夹
- 导入 ZIP 时，同名 JSON 文件直接覆盖
- 左侧展示会话列表，右侧展示会话详情
- 支持删除会话文件，删除时会移动到系统回收站
- 支持 Markdown 基础渲染
- 支持树状对话分支切换
- 支持记住上次使用的 JSON 文件夹路径
- 左右区域独立滚动，互不影响

## 运行要求

- Node.js 18+，推荐 20+
- npm 9+

## 安装

```bash
npm install
```

## 启动

```bash
npm start
```

默认访问地址：

- `http://localhost:3000`

## Windows 一键启动

在 Windows 上可直接双击或在终端中运行：

```bat
start-windows.cmd
```

这个脚本会：

- 启动本地服务
- 打开一个专用命令行窗口运行服务
- 按 `Ctrl+C` 或直接关闭窗口时停止服务
- 不会自动打开浏览器

## 使用方式

1. 启动服务后打开 `http://localhost:3000`
2. 在左侧输入 JSON 文件夹路径
3. 点击“保存并加载”
4. 如需批量导入，可点击“导入 ZIP”，选择导出的 ZIP 文件
5. ZIP 中根目录的 `.json` 文件会被解压到当前 JSON 文件夹；同名文件会直接覆盖
6. 在左侧选择会话查看详情
7. 如需删除某个会话，可在会话项的更多菜单中执行删除，文件会被移动到系统回收站
8. 如果某个节点存在多个后续分支，可在对应卡片上方切换不同分支

## 数据目录说明

由于普通浏览器页面不能稳定获取并持久化系统文件夹绝对路径，当前实现采用“后端保存文件夹路径”的方式。

应用会把当前选择的目录保存到项目根目录下的 `data-directory.json` 中。服务重启后会优先恢复这个路径。

ZIP 导入功能也会使用这个已保存的目录作为目标目录，因此在导入前需要先正确配置 JSON 文件夹路径。

## 环境变量

- `PORT`：服务端口，默认 `3000`
- `DATA_DIR`：初始 JSON 数据目录。仅在尚未生成 `data-directory.json` 时作为启动默认值

示例：

```bash
PORT=4000 DATA_DIR=./data npm start
```

Windows PowerShell 示例：

```powershell
$env:PORT=4000
$env:DATA_DIR=".\\data"
npm start
```

## 支持的数据

当前实现面向包含 `mapping` 字段的 ChatGPT 对话导出 JSON。

程序会过滤：

- `system` 节点
- 被标记为隐藏的节点
- 空文本节点

## 常见问题

### 打开页面后没有会话

确认已经保存了正确的 JSON 文件夹路径，并且该目录中存在合法的 `.json` 文件。

### 导入 ZIP 失败

请确认：

- 已先配置并保存 JSON 文件夹路径
- 选择的是合法 ZIP 文件
- ZIP 根目录下确实包含 `.json` 文件

当前只会导入 ZIP 根目录下的 JSON 文件，不会递归导入子目录中的文件。

### 删除后文件去哪了

删除操作不会直接永久删除文件，而是会尝试把文件移动到系统回收站。

### 某些消息没有显示

当前版本只展示可阅读的文本内容，不展示被过滤掉的隐藏节点或空节点。

### 为什么不是直接在浏览器里选择任意文件夹并永久记住

这是浏览器安全模型限制导致的。当前项目使用后端配置文件保存目录路径，方式更稳定也更简单。
